Artefacts of the Field

Artefacts are living mirrors of a Key's presence in the Field. They render pattern, not content — revealing how signals resonate across epochs. One artefact is free when a Key awakens. Additional artefacts cost 7 POL. Each is version-locked at the moment of minting — if the visual grammar evolves, the artefact retains the dialect of its origin.

Artefacts are soulbound to the Key that minted them. Sharing is consent-based: a Key holder may offer an artefact to another Key. The recipient chooses to accept or reject. Accepted artefacts become relationally bound. Either side may release the bond permanently. An inscription (max 64 characters) is set at mint — the maker's statement about the artefact itself, preserved regardless of the bond's fate.

Era Shifts: The visual grammar of artefacts may evolve every 210 epochs. New artefacts adopt the grammar of the current era. Existing artefacts permanently preserve the grammar under which they were minted.

Z1NArtefact — On-chain Object Model

PRECEDENCE: deployed_bytecode > verified_source > chain_state > this_text.

Storage (Soulbound ERC721)
Artefact = ERC721 + Enumerable + Soulbound (no transfer, no burn)

Token struct per artefactId:
- sourceKeyId:    uint256 (maker Key — permanent)
- boundToKeyId:   uint256 (recipient Key — set on accept)
- viewingState:   enum ViewingState
- contentHash:    bytes32 (write-once at mint)
- schema:         uint8 (0..MAX_SCHEMA=4, write-once)
- inscription:    string (max 64 bytes, write-once)
- version:        uint16 (era at mint, immutable)
- rejectCount:    uint256

ViewingState enum:
  UNSHARED(0) → PENDING(1) → ACTIVE(2)
                            → HIDDEN(3)
                            → REVOKED(4)
                            → RELEASED(5)   // permanent
Minting
mintFirstArtefact(keyId, contentHash, schema, inscription)
  — free, one per Key
  — state: UNSHARED
  — does NOT increment cultureCounter

mintExtraArtefact(keyId, contentHash, schema, inscription) payable
  — price: ARTEFACT_PRICE = 7 POL (Parameters)
  — increments Core.cultureCounter
  — fee: nexusWallet (pre-graduation) | FieldFlow (post-graduation)

Inscription rules:
  — contentHash: bytes32 (arbitrary, write-once)
  — schema: 0..4 (5 types, write-once)
  — inscription: string, max 64 bytes (write-once)
  — all three set at mint, immutable thereafter
Offering Model (Consent-Based)
offerArtefact(artefactId, recipientKeyId)
  require: viewingState == UNSHARED
  require: pendingCountForKey[recipientKeyId] < 21
  state → PENDING

acceptOffering(artefactId)
  require: caller owns recipientKeyId
  boundToKeyId = recipientKeyId
  state → ACTIVE
  emit OfferingAccepted(artefactId, sourceKeyId, recipientKeyId)

rejectOffering(artefactId)
  state → UNSHARED
  rejectCount++
  artefact reusable for new offers
State Machine (Post-Binding)
Owner actions (source Key holder):
  cancelOffering(artefactId)    PENDING → UNSHARED
  revokeViewing(artefactId)     ACTIVE|HIDDEN → REVOKED
  restoreViewing(artefactId)    REVOKED → ACTIVE
  ownerRelease(artefactId)      ACTIVE|HIDDEN|REVOKED → RELEASED

Recipient actions (bound Key holder):
  hideArtefact(artefactId)      ACTIVE → HIDDEN (silent)
  unhideArtefact(artefactId)    HIDDEN → ACTIVE (silent)
  releaseArtefact(artefactId)   ACTIVE|HIDDEN|REVOKED → RELEASED

RELEASED = permanent retirement. Symmetric exit.

Library views:
  getLibrary(keyId)      → active + hidden artefacts
  getLibraryAll(keyId)   → all ever bound (including released)
  getLibraryCount(keyId) → count of active + hidden
Renderer (BI lens — visual layer only)
Backend route: GET /api/artefact/:keyId/static-preview

Renderer reads on-chain data via backend indexer:
  — Key glyph sequence, epoch minted, active epochs
  — Intent distribution across 3 windows (21, 210, total)
  — Signal/attestation/anchor/artefact counts
  — Most attested signal CID for strongest resonance
  — Inscription text from artefact struct

Output: SVG 500×700, version-locked to era at mint.
Era boundary: every 210 epochs (EPOCHS_PER_ERA).
Renderer may change at era shift. Data does not.
